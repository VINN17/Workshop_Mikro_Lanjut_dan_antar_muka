‚öôÔ∏è Slave Logic: State Machine

The slave uses an interrupt-driven state machine to handle SPI data without blocking the main loop.

[Image of finite state machine diagram]
switch (state) { case WAIT_SYNC1: if (input == 0xFF) state = WAIT_SYNC2; break; case WAIT_ID: if (input == MY_ID) state = WAIT_CMD; else state = WAIT_SYNC1; break; case WAIT_CS: if (input == calc_cs) sendResponse(); break; }

States:

  1. WAIT_SYNC1: Waiting for 0xFF start byte.
  2. WAIT_SYNC2: Waiting for 0xFE confirmation.
  3. WAIT_ID: Checks if the packet is addressed to this slave.
  4. WAIT_CMD: Stores the command type.
  5. WAIT_CS: Validates integrity before responding.
